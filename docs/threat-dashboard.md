# Threat Dashboard

The Threat Dashboard is a full-page table view of every threat across your project — across all saved threat models and diagram versions. Use it to track remediation status, filter by severity or STRIDE category, and export a PDF report.

> Navigate to the dashboard: **Toolbar → Threat Model → Dashboard**, or visit `/projects/<id>/threats`.

---

## Summary Bar

At the top of the page, four counters give an at-a-glance view of your project's threat posture:

| Counter | Description |
|---------|-------------|
| **Total active** | All threats not marked as False Positive |
| **Critical** | Threats with CRITICAL severity |
| **High** | Threats with HIGH severity |
| **Mitigated** | Threats marked Mitigated or Accepted |

These counts ignore current filter state — they always reflect the full project.

---

## Search and Filter

Use the controls above the table to narrow what you see:

| Control | What it filters |
|---------|----------------|
| **Search box** | Matches threat title, description, or target node name |
| **Severity** dropdown | CRITICAL / HIGH / MEDIUM / LOW / INFO |
| **Status** dropdown | Identified / In Progress / Mitigated / Accepted / False Positive |
| **STRIDE** dropdown | S / T / R / I / D / E category |

Filters combine — e.g., `Severity: CRITICAL` + `Status: Identified` shows only unaddressed critical threats.

---

## Threat Table Columns

| Column | Description |
|--------|-------------|
| **Title** | Threat name |
| **STRIDE** | Category badge (color-coded) |
| **Severity** | CRITICAL / HIGH / MEDIUM / LOW / INFO badge |
| **Status** | Current remediation status (editable inline) |
| **Target** | Node or edge name the threat applies to |
| **Source** | AI (generated by analysis) or User (manually added) |
| **Model** | Which saved threat model this threat belongs to |
| **Version** | Diagram version at the time of analysis |
| **Actions** | View, Edit, Dismiss, Delete |

---

## Change Threat Status

Click the **Status dropdown** directly in the table row to update a threat's status without opening a detail view:

- **Identified** → In Progress → Mitigated (linear workflow)
- **Accepted** — risk accepted, no fix needed
- **False Positive** — hides the threat from active counts

Status changes are saved immediately to the backend.

---

## Add a Threat Manually

Click **+ Add Threat** in the filter bar to open the Add Threat modal:

1. Select the **threat model** to add it to (must be a saved model).
2. Fill in:
   - Target node or edge
   - STRIDE category
   - Severity
   - Title and description
   - Optional mitigation notes
3. Click **Save**.

Manually added threats are tagged with a **User** source badge.

---

## Pagination

The dashboard shows **20 threats per page**. Use the `← 1 2 3 →` controls at the bottom to navigate. The page number is preserved when you change filters.

---

## Export PDF Report

Click **Export Report** in the bottom-right corner to download a PDF threat model report.

The PDF includes:
- **Cover page** — project name, diagram version, analysis date, and security score
- **Executive summary** — threat count by severity, percentage mitigated
- **Threat catalog** — all threats grouped by STRIDE category
- **Mitigation status table** — all threats with their current status and notes

The report is suitable for security reviews, SOC2 / ISO27001 audits, and stakeholder presentations.

> The export uses the latest saved threat model data. Run and save a new STRIDE analysis before exporting if your diagram has changed.

---

## Navigate to a Threat on the Canvas

Click the **View** (eye) icon on any threat row to jump back to the diagram editor with the relevant node highlighted. Use this to quickly locate a node described in a threat and make architectural changes.

---

## Delete a Threat Model

Threats in the dashboard belong to saved threat models. To delete an entire saved threat model (and all its threats):

1. Open the **Threat Model Panel** (`Cmd+Shift+M`).
2. Open the **History** tab.
3. Click **Delete** next to the model you want to remove.

Individual threats can be deleted from the dashboard using the **Delete** action on a row.
